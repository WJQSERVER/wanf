# Neo: 实验性超高性能编解码器

Neo 是 WANF 库中提供的一套实验性、追求极致性能的编解码实现。它通过使用 `unsafe` 指针直接操作内存、最小化反射开销以及深度优化的词法分析器，实现了近乎零分配（Zero-Allocation）的序列化与反序列化。

## 设计目标

- **极致性能**：在处理大型或复杂配置文件时，提供比标准编解码器更快的速度。
- **低内存分配**：通过 `sync.Pool` 复用资源和 `unsafe` 转换，极大地减少 GC 压力。
- **原生支持**：直接支持 Go 结构体，无需中间 AST 转换。

## 使用方法

Neo 提供了简单的包级函数供直接调用：

```go
import "github.com/WJQSERVER/wanf"

// 序列化
data, err := wanf.NeoMarshal(&config)

// 反序列化
err := wanf.NeoUnmarshal(data, &config)
```

> **注意**：`NeoMarshal` 要求传入的必须是可取地址的指针类型，以实现零拷贝的字段访问。

## 支持的特性

- **基本类型**：`string`, `int`, `int64`, `float64`, `bool`。
- **结构体 (Block)**：支持嵌套结构体，映射为 WANF 中的块语法。
- **指针类型**：支持指向基本类型或结构体的指针。反序列化时会自动为 `nil` 指针分配内存。
- **持续时间**：原生支持 `time.Duration`，支持时长字符串（如 `"10s"`, `"-5m"`）和纳秒整数。
- **标签支持**：完整支持 `wanf` 结构体标签，包括自定义字段名和 `omitempty` 选项。
- **兼容性匹配**：解码时支持大小写不敏感的字段名匹配（作为精确匹配失败后的回退方案）。

## 性能表现

在基准测试中，Neo 表现出了显著的优势：

- **词法分析**：相比标准 Lexer，NeoLexer 的分配次数显著降低。
- **编解码**：对于简单结构，可以达到 0 次内存分配；对于复杂结构，分配次数也维持在极低水平（通常仅为 1-2 次）。

## 局限性与警告

由于 Neo 仍处于实验性阶段且为了追求极致性能进行了一些权衡，它存在以下局限：

1. **功能受限**：目前不支持变量替换 (``)、环境变量 (`env()`) 和文件导入 (`import`)。
2. **集合支持**：目前对 `slice` 和 `map` 的支持尚不完整（正在开发中）。
3. **安全性**：内部大量使用 `unsafe`，虽然经过了严格测试，但在极端边界情况下需谨慎使用。
4. **错误处理**：为了性能，错误报告可能不如标准编解码器详细。

如果您的应用场景对配置文件的解析性能有极高要求，且不需要上述高级功能，Neo 是一个理想的选择。
