# Neo: 极致性能编解码器

Neo 是 WANF 库中提供的一套追求极致性能的编解码实现。它通过使用 `unsafe` 指针直接操作内存、最小化反射开销以及深度优化的词法分析器，实现了极高的吞吐量和极低的内存分配。

## 设计目标

- **极致性能**：在处理大型或复杂配置文件时，提供显著优于标准编解码器的速度（通常有 3 倍以上的提升）。
- **低内存分配**：通过 `sync.Pool` 复用资源、`unsafe` 转换以及直接内存写入，将分配次数降至最低。
- **全量语法支持**：在保持高性能的同时，实现了与标准解码器完全一致的 WANF 语法支持。

## 使用方法

Neo 提供了简单的包级函数供直接调用：

```go
import "github.com/WJQSERVER/wanf"

// 序列化
data, err := wanf.NeoMarshal(&config)

// 反序列化
err := wanf.NeoUnmarshal(data, &config)
```

> **注意**：为了实现零拷贝的字段访问，传入 `NeoMarshal` 和 `NeoUnmarshal` 的必须是可取地址的指针类型。

## 支持的特性

- **基本类型**：`string`, `int`, `int64`, `float64`, `bool`。
- **集合类型**：完整支持 `slice` (切片) 和 `map` (映射)。针对常用类型如 `[]string`, `[]int`, `map[string]string` 提供了高度优化的快速路径。
- **结构体 (Block)**：支持嵌套结构体，映射为 WANF 中的块语法。
- **高级语法**：
    - **变量支持**：支持 `var` 声明及 `${variable}` 引用。
    - **文件导入**：支持 `import` 语句（带内部缓存优化）。
    - **动态求值**：支持 `env()` 环境变量读取。
- **指针类型**：支持指向基本类型或结构体的指针。反序列化时会自动为 `nil` 指针分配内存。
- **持续时间**：原生支持 `time.Duration`，支持时长字符串（如 `"10s"`, `"-5m"`）和纳秒整数。
- **标签支持**：完整支持 `wanf` 结构体标签，包括自定义字段名和 `omitempty` 选项。
- **兼容性匹配**：支持线性探测哈希表加速的字段查找，并支持大小写不敏感匹配作为回退方案。

## 性能表现

在基准测试中，Neo 展现了统治级的优势：

- **词法分析**：通过非流式快速路径和查找表优化，NeoLexer 的速度极快且分配为零。
- **复杂解码**：在包含大量嵌套、变量和导入的复杂配置中，Neo 的速度是标准解码器的 3 倍以上，且内存分配极少。
- **简单结构**：对于简单结构，可以实现 0 次内存分配。

## 局限性与警告

1. **安全性**：内部大量使用 `unsafe` 以压榨性能。虽然经过了严格的单元测试验证，但在安全性要求极高的极端环境下请根据实际情况选择。
2. **错误处理**：为了极致性能，错误报告的详细程度可能略逊于标准编解码器。

如果您的应用场景对配置文件的解析性能有极高要求，Neo 是最佳选择。
